1. Product Overview & Vision
Deslint is the first quality gate built for how code looks, not just how it runs. It validates design quality — spacing consistency, typography hierarchy, color token adherence, responsive coverage, and accessibility — in AI-generated frontend code across any framework.
Product Pillars
1. Local-First: Zero code leaves the developer's machine. ESLint runs locally. MCP server uses stdio. No cloud dependency.
2. Framework-Agnostic: Works with React, Vue, Svelte, Angular, plain HTML — any Tailwind or CSS-based output.
3. AI-Tool Agnostic: Validates output from Claude Code, Cursor, v0, Bolt, Lovable, Copilot — any AI coding tool.
4. Open Core: Free OSS ESLint plugin for adoption. Paid CLI, MCP server, and dashboard for revenue.
Tech Stack
Language: TypeScript (everything)
ESLint Plugin: @deslint/eslint-plugin — AST-based rules via @typescript-eslint/utils
CLI: @deslint/cli — Commander.js, Chalk, project-wide analysis + Design Health Score
MCP Server: @deslint/mcp — @modelcontextprotocol/sdk, stdio transport, local execution
Dashboard: Next.js 15 + Tailwind + shadcn/ui (Framer for marketing site)
API: Next.js API routes + Drizzle ORM + PostgreSQL (Supabase)
Monorepo: Turborepo with packages/ structure
CI/CD: GitHub Actions
Hosting: Vercel (dashboard) + npm (packages)

3. Definition of Done
Every user story must meet ALL criteria before marking complete:
	•	Code written in TypeScript with strict mode enabled
	•	Unit tests written and passing (minimum 80% coverage for new code)
	•	Integration tests passing for cross-module functionality
	•	No TypeScript errors (tsc --noEmit passes)
	•	ESLint passes on own codebase (dogfooding)
	•	PR created with description, linked to user story ID
	•	Self-review completed (CTO hat review checklist)
	•	Security review for any external input handling
	•	Documentation updated (README, JSDoc, CHANGELOG)
	•	Business acceptance: CEO hat confirms story delivers stated value
	•	McKinsey check: Does this move the product toward revenue?
