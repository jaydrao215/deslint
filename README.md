# Vizlint

**The design quality gate for AI-generated frontend code.**

Vizlint validates how your code *looks*, not just how it runs. It catches spacing inconsistencies, typography drift, arbitrary colors, missing responsive breakpoints, and accessibility issues — in code generated by Claude Code, Cursor, v0, Bolt, Lovable, or any AI coding tool.

## Quick Start

```bash
# Install the ESLint plugin
npm install eslint-plugin-vizlint --save-dev
```

```js
// eslint.config.js (ESLint v10+ flat config)
import vizlint from 'eslint-plugin-vizlint';

export default [
  vizlint.configs.recommended,
  // ... your other configs
];
```

## Rules

| Rule | Description | Status |
|------|-------------|--------|
| `vizlint/no-arbitrary-colors` | Flag hardcoded hex colors in Tailwind classes | 🚧 Sprint 1 |
| `vizlint/no-arbitrary-spacing` | Flag hardcoded spacing values | 📋 Sprint 2 |
| `vizlint/typography-scale` | Enforce consistent type scale | 📋 Sprint 2 |
| `vizlint/responsive-required` | Flag fixed-width layouts without breakpoints | 📋 Sprint 3 |
| `vizlint/consistent-component-spacing` | Detect spacing divergence across components | 📋 Sprint 3 |
